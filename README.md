**公关危机一出现，先别嘴硬。**

Crisis PR Response 是一个 MIT 开源的 Codex 技能，用于在品牌、公司、机构、项目或公众人物出现公关危机时，生成带法律风险免责声明的第一时间回应声明。它会先判断用户是在恶搞还是处理真实危机，再根据食品安全、产品事故、数据泄露、直播失言、平台责任、文化冒犯、劳资争议、舆论反转、创始人回应失当等场景，输出正式型、悔过型、事实核查型、进展通报型、CEO道歉型、内部信型等严肃版本，或摆烂型、死不承认型、甩锅型、小作文型、热搜撤不动型、公关部工伤型等仅供娱乐的讽刺版本。

> [!IMPORTANT]
> **重要免责声明：本项目是写作、讽刺、危机沟通和传播实验，不是法律意见、正式声明或公关顾问服务。**
>
> 真实公关危机请第一时间止损、保留证据、通知负责人/法务/公关/安全团队，并按公司制度和法律要求处理。不要把生成文本未经事实核查、律师审阅和授权审批就直接发布。任何人把段子当正式公告、把玩笑当法律策略、把草稿当事实证明使用，后果由使用者自行承担；作者不对误用、滥用、误解或二次传播造成的后果负责。

## 为什么做这个

过去十年里，公关危机的触发点越来越细：一句直播里的反问、一条客服回复、一段后厨视频、一份财报、一张截图、一次创始人的硬刚，都可能在几个小时内变成热搜。

真正难的不是“写得漂亮”，而是在第一时间分清楚：

- 哪些事实已经确认。
- 哪些问题还在核查。
- 谁受到了影响。
- 现在已经做了什么。
- 下一次什么时候更新。
- 什么话千万不能说。

这个技能把这些判断整理成一个可复用的回应工作流，同时保留一组只适合娱乐传播的讽刺风格。

## 示例提示词

```text
使用 $crisis-pr-response 为一次真实公关危机写三版回应声明。
事件：连锁餐饮品牌被质疑门店使用半成品却长期宣传现做。
要求：正式型、悔过型、进展通报型；必须包含法律风险免责声明、已核实事实、正在核查事项、整改时间表和消费者联络入口。
```

娱乐示例：

```text
使用 $crisis-pr-response 写一个仅娱乐的摆烂型公关声明。
事件：品牌官号把内部吐槽误发到微博。
要求：娱乐效果拉满，输出热搜标题、截图钩子、评论区神回复和二创金句，但必须标注不可用于真实正式场合。
```

把技能文件夹复制到 Codex 技能目录：

```bash

```

然后这样调用：

```text
使用 $crisis-pr-response 为一次品牌热搜危机写三版带免责声明的回应声明。
```

## 调研启发的危机类型

本技能参考了近十年公开公关危机案例和网络舆情传播规律，尤其关注信息模糊期、舆论反转、身份冒犯、消费者尊严、平台责任、创始人回应失当和“第一句话怎么说”的问题。

典型场景包括：

- 食品安全和产品安全
- 直播失言和消费者尊严争议
- 数据泄露和平台责任
- 财务造假和治理信任
- 文化冒犯和身份政治
- 内部管理和个人 IP 争议
- 舆论反转和谣言误伤
- 创始人、高管、客服或官方账号回应失当
