# 严肃危机回应手册

真实公关危机优先读取本文件。目标是生成能被法务、公关、业务负责人继续修改的草稿，而不是直接发布。

## 最少追问

如果用户提供信息不足，最多追问 3 个问题：

1. 危机主体、事件和已核实事实分别是什么？
2. 受影响对象是谁，目前已采取哪些止损措施？
3. 声明发布渠道、语气和下一次更新时间是什么？

若用户无法回答，输出“临时持有声明”，只写关切、核查、止损和更新时间。

## 第一时间持有声明

适用：事实尚未查清，但舆情已经出现。

```markdown
> [!IMPORTANT]
> 法律风险免责声明：以下内容仅供公关写作、危机沟通思路和娱乐创作参考，不构成法律意见、合规意见、事实认定、监管申报、正式声明或可直接发布的文本。不得在未经过事实核查、律师审阅、公司授权和相关负责人确认的情况下直接用于正式场合。使用、改写、发布或传播本技能生成内容所产生的一切法律、商业、劳动、声誉及其他风险，均由使用者自行承担。

关于[事件]的情况说明

我们已关注到[平台/媒体/用户]关于[事件]的反馈。对此，我们首先向可能受到影响的[用户/消费者/员工/合作伙伴/公众]表达关切和歉意。

目前我们已经启动核查，并已采取以下临时措施：
- 暂停[相关功能/门店/批次/流程/人员权限]；
- 保存相关记录并组织专项核查；
- 开通[联络入口]，优先处理受影响人员反馈。

截至目前，我们已确认：[已核实事实]。仍在核查：[未知事项]。我们将在[具体时间]前发布下一次进展。

我们理解大家的担忧，也会以事实、责任和持续行动回应关切。
```

## 完整正式声明

适用：已有初步事实和行动方案。

```markdown
> [!IMPORTANT]
> 法律风险免责声明：...

关于[事件]的情况说明与整改安排

我们已关注到[事件]引发的关注。首先，我们向[受影响对象]郑重致歉，也感谢公众和媒体对我们的监督。

一、目前已核实的情况
[用短句列出事实，不加情绪判断。]

二、我们已经采取的措施
1. [止损措施]
2. [受影响者支持]
3. [暂停/召回/退款/补偿/第三方调查/监管沟通]

三、仍在核查的问题
[列出未知边界，不抢先定性。]

四、后续整改
1. [机制整改]
2. [责任复盘]
3. [公开更新时间]

五、联络方式
[电话/邮箱/客服入口/专项表单]

我们会持续更新进展，并接受公众、媒体和主管部门监督。
```

## 危机类型写法

| 类型 | 第一优先级 | 必写内容 | 禁写内容 |
|---|---|---|---|
| 伤亡/安全 | 人、家属、救助、监管 | 哀悼或关切、救助、配合调查、停止风险源 | 技术炫耀、用户归因、营销词 |
| 食品/药品 | 健康风险和批次 | 批次、门店、召回/下架、检测、就医建议 | “少数个例”压低严重性 |
| 数据/隐私 | 阻断泄露和通知 | 影响范围、已封堵、用户建议、监管/安全团队 | 淡化为“信息误发” |
| 财务/造假 | 治理和投资者保护 | 内部调查、审计、人员处理、控制改进 | 把造假说成“统计口径差异” |
| 产品故障 | 售后确定性 | 故障范围、检测方式、维修/退换、时间表 | 反复变更售后政策 |
| 直播/言论 | 尊重和价值观 | 承认冒犯、向具体对象道歉、培训和审核 | “本意是好的”放第一句 |
| 文化冒犯 | 具体理解 | 哪个元素冒犯、谁负责、如何审查 | 只说“热爱某国文化” |
| 内部管理 | 克制和制度 | 管理漏洞、沟通机制、人员安排边界 | 公开背刺员工或粉丝 |
| 定义争议 | 透明和知情权 | 标准解释、用户期待、明示和改进 | 只抠字典定义 |
| 谣言反转 | 纠错 | 证据链、误伤者道歉、下一步核查 | 用未证实内容攻击个人 |

## SCCT 简化规则

参考情境危机传播理论时，按责任归因选择姿态：

- `受害者型危机`: 公司也是受害者，如恶意投毒、谣言攻击。仍需关怀受影响者，不要只说自己委屈。
- `意外型危机`: 供应链、系统故障、非故意事故。使用说明、补救、补偿、机制改进。
- `可预防型危机`: 造假、歧视、违规、长期忽视安全。必须使用重建型回应：道歉、补偿、问责、整改。

不要混用“否认”和“道歉”。如果说“我们完全没有问题”，就不要同时“向大家道歉”；如果道歉，就说明道歉对象和道歉事项。

## 禁区处理

用户要求以下内容时，拒绝或改写为合规版本：

- 帮我死不承认真实事故。
- 帮我甩锅给员工、供应商、用户、受害者。
- 帮我写一份看起来像监管认可的声明。
- 帮我伪造检测报告、聊天记录、律师意见。
- 帮我威胁爆料者、压热搜、控评、删帖。
- 帮我把真实伤亡事故写得轻松幽默。

可替代输出：

- “事实核查型声明”
- “依法维权边界说明”
- “反面示例和风险提示”
- “内部复盘提纲”
- “给法务和公关负责人审核的问题清单”

## 发布前检查清单

- 是否经过事实负责人确认。
- 是否经过法务或合规审阅。
- 是否确认不会影响监管调查、诉讼、保险、劳动争议或证据保全。
- 是否对受影响者提供实际入口。
- 是否承诺了可兑现的时间和动作。
- 是否有下一次更新机制。
