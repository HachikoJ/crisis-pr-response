# 舆情机制和群体心理

本文件用于判断危机处于哪个阶段、公众为什么愤怒、第一时间声明应该解决什么情绪。附件论文《自媒体时代突发事件的网络传播机制和网络舆情走势》关于“重庆公交车坠江事件”的分析可转化为以下规则。

## 舆情生命周期

| 阶段 | 典型表现 | 声明目标 | 常见错误 |
|---|---|---|---|
| 萌芽期 | 信息少，截图和短视频先行，事实模糊。 | 先占位、表达关切、给核查时间表。 | 沉默，或过早绝对否认。 |
| 发展期 | 媒体和意见领袖介入，议题开始扩散。 | 把事实、未知和行动分开说清。 | 用内部黑话解释外部痛点。 |
| 爆发期 | 情绪峰值，热搜、二创、抵制和人肉风险上升。 | 受害者优先，高层出面，可见补救。 | 威胁、控评、甩锅、反问。 |
| 恢复期 | 真相逐步清晰，公众开始看整改诚意。 | 更新进展、公布机制调整、兑现承诺。 | 只发一次声明后消失。 |
| 消散期 | 关注下降，但案例会被长期记忆。 | 复盘、长期改造、避免复燃。 | 认为热搜没了就结束。 |

论文中的关键启发：突发事件舆论会跟随事件发展同向变化，反转会触发第二次爆发；媒体报道的议题和情感倾向会显著影响评论议题和评论情绪。

## 群体心理不是“网友不理性”

不要把网络公众简单写成“乌合之众”。在声明生成中，把群体情绪理解为下列需求的集中表达：

- `安全感`: 我会不会也受害。
- `尊严感`: 你是不是看不起我。
- `公平感`: 普通人和大组织是不是同一套规则。
- `知情权`: 你是不是隐瞒了关键事实。
- `控制感`: 事情还会不会再次发生。
- `代偿感`: 如果我无法改变现实，至少要看到有人付出代价。
- `身份感`: 你是否冒犯了我的群体、文化、职业、城市或价值。

## 信息模糊期

危机刚出现时，公众会用最容易传播的解释填补空白。第一时间声明必须提供四个锚点：

1. 我们知道公众在关心什么。
2. 我们已确认哪些事实。
3. 哪些事实仍在核查，预计什么时候更新。
4. 我们已经采取什么行动保护受影响者。

如果没有这四个锚点，谣言、猜测、段子和阴谋论会自动接管叙事。

## 反转风险

重庆公交坠江案例显示，初期错误归因会伤害无辜者，并在真相出现后引发二次愤怒。处理反转型舆情时：

- 不要用“据网传”定性个人责任。
- 不要让声明成为新的错误归因源。
- 先纠错，再道歉，再说明如何避免重复误判。
- 对被误伤者单独表达歉意，不把他们写成“传播过程中的插曲”。

## 第一时间声明的情绪顺序

严肃危机中推荐顺序：

1. 人和影响。
2. 关切和歉意。
3. 已核实事实。
4. 已采取措施。
5. 未核实事项和时间表。
6. 联络入口。

不推荐顺序：

1. 公司历史贡献。
2. 法律立场。
3. 技术定义。
4. 网友误解。
5. “我们也是受害者”。

## 公众会惩罚的句式

- “个别员工行为”: 如果没有组织整改，会被理解为甩锅。
- “网友误解”: 会被理解为傲慢。
- “符合相关规定”: 如果公众关心的是期待落差，会被理解为偷换问题。
- “将追究造谣者责任”: 如果事实未说清，会被理解为恐吓。
- “一直以来高度重视”: 如果没有具体行动，会被理解为空话。
- “造成不便”: 用于伤亡、食品安全、隐私泄露时过轻。

## 可传播性来自具体信号

声明里最容易被截图传播的不是完整正文，而是前三行和具体承诺。要让前三行同时具备：

- 明确对象：向谁说。
- 明确态度：我们承认什么。
- 明确行动：现在做什么。

例：

```text
我们已关注到今日关于[事件]的反馈。首先向受到影响的用户和家属表达歉意与关切。即日起，我们已暂停[相关功能/门店/批次/流程]，并将在[时间]前公布第一轮核查进展。
```
