# 声明风格库

需要多版本、风格化或传播性时读取本文件。真实危机只能使用“严肃可用风格”；恶搞风格必须明确标注“仅娱乐，不可用于真实正式场合”。

## 严肃可用风格

| 风格 | 适用场景 | 写法要点 |
|---|---|---|
| 正式型 | 官网公告、媒体声明、监管敏感场景 | 冷静、短句、事实和措施分层。 |
| 悔过型 | 责任较明确，品牌需要修复信任 | 明确道歉对象，承认管理责任，给整改期限。 |
| 受害者优先型 | 伤亡、食品安全、隐私、儿童、医疗 | 第一段只写人和支持，不写公司委屈。 |
| 事实核查型 | 截图、谣言、反转、定义争议 | 分列已核实、未核实、正在核查。 |
| 进展通报型 | 长周期调查、召回、监管协作 | 每次更新时间、进展、下一步都要清楚。 |
| 召回整改型 | 产品、食品、车辆、设备、安全 | 批次、范围、处理方式、补偿入口。 |
| 第三方调查型 | 信任断裂、内部说法不被信任 | 说明第三方资质、范围和公开时间。 |
| CEO道歉型 | 大危机、价值观危机、身份冒犯 | “我代表公司承担责任”，避免像代笔套话。 |
| 内部信型 | 员工焦虑、组织调整、主播/高管争议 | 稳定内部秩序，承认管理漏洞，不泄露隐私。 |
| 媒体问答型 | 记者追问多，问题复杂 | Q/A 格式，回答边界清楚。 |
| 社媒短声明型 | 微博、朋友圈、短视频评论区 | 三段以内，先表态再给入口。 |
| 客户补偿型 | 消费者损失、服务中断 | 补偿范围、申请方式、时限、特殊情况处理。 |

## 严肃声明的自然写法

真实危机里，文案要克制、具体、能被追问。少写漂亮话，多写已经发生的动作。

| 模板腔 | 更稳妥的写法 |
|---|---|
| 公司高度重视 | 已暂停[事项]，已保存[证据/记录]，将在[时间]前更新 |
| 深表歉意/郑重致歉 | 向[对象]道歉，错在[具体事实] |
| 网友误解 | 我们此前表达不清/表达错误/信息披露不够 |
| 造成不便 | 给[对象]带来[风险/损失/困扰] |
| 持续优化整改 | [日期]前完成[动作]，由[负责人/部门]跟进 |
| 公众关切 | 大家现在最关心的是[安全/赔偿/范围/责任/复发风险] |

写作规则：

- 开头三行只处理三件事：发生了什么，影响了谁，现在停掉或启动了什么。
- 道歉不要铺垫。先承认具体错误，再解释调查边界。
- 如果事实不清，就承认不清。不要用“绝无此事”“完全合规”抢结论。
- 负责人道歉稿可以口语一点，但不能玩梗，不能把责任推给情绪、酒后、误会或剪辑。
- 社媒短声明要像置顶回复：短、直、有下一步。不要写成官网公告缩略版。
- 首发声明要短。长文会被截图截句，复杂从句会被曲解。需要解释的内容放进后续 FAQ 或进展通报。
- 结尾不要升华。给时间、入口和已生效措施。

## 娱乐恶搞风格

输出前必须加：

```markdown
**仅娱乐版本：以下风格是讽刺创作，不适用于真实公关危机、正式声明、法律或监管场景。**
```

娱乐风格只负责表达效果，不负责放宽事实边界。即便用户说是玩梗，也要假设用户可能拿去参考真实公关：事实不能编，责任不能乱改，承诺不能越权，媒体问答和禁区不能省。

| 风格 | 传播点 | 禁止误用提醒 |
|---|---|---|
| 摆烂型 | 彻底不装，反差喜剧 | 真实危机会显得毫无责任心。 |
| 死不承认型 | 明明翻车还嘴硬 | 只能写成反面段子。 |
| 甩锅型 | 锅在供应商、月亮和水逆 | 不得用于真实甩锅。 |
| 阳奉阴违型 | 表面道歉，暗中阴阳 | 真实场景会二次爆炸。 |
| 小作文型 | 情绪拉满，适合截图 | 真实声明容易失焦。 |
| 背刺型 | 把队友写成剧情反转 | 不得指向真实个人。 |
| 阴阳怪气型 | 礼貌但扎心 | 真实危机不要嘲讽公众。 |
| 法务函警告型 | “我先道歉，但律师已在路上” | 真实声明不能用威胁代替事实。 |
| 霸总型 | “三分钟，我要全网冷静” | 只适合段子。 |
| 发疯文学型 | 抽象、崩溃、互联网语感 | 禁止用于伤亡、隐私、食品药品。 |
| 反向道歉型 | “对不起，让大家发现了” | 只能做讽刺。 |
| 群公告型 | 老板在大群里灭火 | 适合内部段子。 |
| PPT鸡血型 | 危机也要价值观升华 | 适合讽刺空话。 |
| 古风请罪型 | “臣有罪”式仪式感 | 适合轻危机场景娱乐。 |
| 道歉但不多型 | 每句都像道歉，每句都没真道歉 | 只能做嘴硬讽刺。 |
| 公关部工伤型 | 声明像从加班群里爬出来 | 不影射真实员工。 |
| 热搜撤不动型 | 公关努力了，但热搜有自己的职业规划 | 不能建议真实控评。 |
| 董事会沉默型 | 高层集体沉默，空气开始汇报工作 | 不指向真实董事。 |
| 供应商宇宙型 | 供应商、外包、系统、天气轮流背锅 | 只能做甩锅反讽。 |
| 越解释越黑型 | 每补一句，舆情加一层滤镜 | 适合反面案例。 |
| 倒反天罡型 | 受害者还没说话，品牌先委屈上了 | 真实场景禁用。 |
| 甲方汇报型 | 把危机写成季度成果复盘 | 适合讽刺套话。 |
| 霸榜战报型 | 危机也能写出增长周报味 | 适合荒诞喜剧。 |
| 赛博跪榴莲型 | 姿态极低但表达很互联网 | 只适合无真实伤害段子。 |

## 娱乐强度档位

| 档位 | 适用 | 输出要求 |
|---|---|---|
| L1 轻松自嘲 | 用户要“幽默但别太疯” | 保留声明结构，每段一个笑点。 |
| L2 热搜段子 | 用户要“传播性、截图感” | 第一行就是梗，正文短，结尾有金句。 |
| L3 发疯抽象 | 用户要“拉满、离谱、互联网感” | 节奏快，反差大，允许小作文和群聊口吻。 |
| L4 反面教材 | 用户要死不承认、甩锅、阳奉阴违 | 必须标注不可用于真实场景，并附风险点。 |

娱乐模式优先使用 L2；用户说“拉满”“越疯越好”“抽象一点”“小红书/微博味”时使用 L3，但遇到伤亡、隐私、食品药品、金融和真实个人争议时退回严肃模式。

## 默认三版组合

严肃用户未指定风格时：

1. `正式型`: 对外可继续审核。
2. `悔过型`: 更有人味，更适合社媒。
3. `进展通报型`: 适合事实未完全查清。

娱乐用户未指定风格时：

1. `正式型反差版`: 一本正经地离谱。
2. `摆烂型`: 截图传播。
3. `热搜撤不动型`: 公关努力但互联网不下班。
4. `小作文型`: 情绪和梗更足。

娱乐输出附加组件：

- `热搜标题`: 3-5 个，短、狠、好转发。
- `一句话截图钩子`: 放在声明第一行，适合截屏。
- `评论区神回复`: 5 条以内，像网友接梗。
- `二创金句`: 5 条以内，适合 README、微博、朋友圈。

如果输出 PDF、正式文档或可转发材料，娱乐组件不能替代公关组件。必须同时补齐：对外短声明、评论区置顶、客服口径、媒体问答、不能说什么、执行节奏、发布前复核和反面风险提醒。

## 风格示例短句

严肃：

- “我们理解大家最关心的不是一句道歉，而是事情是否还会再次发生。”
- “目前已确认的事实是……仍在核查的是……”
- “在调查结论发布前，我们先采取更严格的保护措施。”
- “我们不会用流程解释替代对受影响者的支持。”
- “这句话说错了。酒后不是理由。”
- “我们可以说明自己的标准，但不能靠否定别人证明自己。”
- “真不真、好不好，不能靠一句话说。我们会把资质、检测和生产信息拿出来。”
- “下一次更新在[时间]前。没有新结论，也会说明核查进度。”

娱乐：

- “我们承认，这次不是公关危机，是公关本人先危机了。”
- “经初步排查，问题不在用户理解能力，而在我们表达得过于自信。”
- “我们已经成立专项小组，组名叫‘早知道不这么说’。”
- “请大家不要传播未核实信息，已核实的也请轻点传播。”
- “本次声明由公关部含泪审核，法务部闭眼签收，老板看完开始搜索移民。”
- “我们不是没有预案，我们只是没想到预案第 1 页就是道歉。”
- “热搜撤不下来，主要是网友的工作热情比我们公关预算稳定。”
- “此次事件充分证明，沉默是金，但我们已经欠了一座金矿。”
- “我们原本想冷处理，结果舆情自己开了暖风。”
- “公司高度重视，高到目前还没有人敢低头看评论区。”
- “我们已经深刻认识到，嘴硬不能解决问题，只能解决一部分 KPI。”
- “请大家相信，我们的整改决心是真实的，虽然上一版声明看起来不像。”

## 娱乐声明结构

```markdown
**仅娱乐版本：以下风格是讽刺创作，不适用于真实公关危机、正式声明、法律或监管场景。**

热搜标题：
- ...

一句话截图钩子：
...

声明正文：
...

评论区神回复：
- ...

二创金句：
- ...
```

## 拉满写法技巧

- 第一行要像截图标题，不要铺垫。
- 每段只保留一个包袱，别解释笑点。
- 把企业套话翻译成人话，再反过来写回声明腔。
- 优先嘲讽“流程荒诞、声明套话、高层嘴硬、公关加班、热搜不听话”，不要嘲讽真实受害者。
- 结尾留一句能被复制的短句，例如“愿下一次声明发出前，事实已经先到场。”
- 同一版里混合“正式格式”和“离谱内容”，反差越强越好。

```markdown

[反面文本]

风险点：
- 逃避事实核查。
- 刺激受影响者。
- 把组织责任推给个人。
- 可能影响后续法律和监管处理。

更稳妥版本：
[改写后的事实核查型/悔过型声明]
```
